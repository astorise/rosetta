## Purpose

This specification defines the requirements for a dual-routing system for processing documents. It details how different worker services (`worker-pdf`, `worker-html`, `worker-jar`) should handle teacher-model outputs, persist vectorized markdown to a vector database (LanceDB), and selectively contribute high-value instructional pairs to a reinforcement learning (RL) dataset.
## Requirements
### Requirement: Document workers consume a structured dual-routing teacher payload

The system MUST require `worker-pdf`, `worker-html`, and `worker-jar` to consume teacher-model output as a structured JSON object containing `rag_markdown`, `pedagogical_value`, and an optional `rl_pair`.

#### Scenario: A worker receives a valid dual-routing teacher payload
- **WHEN** one of the document workers finishes the teacher-model step for a supported source document
- **THEN** it parses `rag_markdown` as the canonical Markdown artifact for downstream routing
- **AND** it evaluates `pedagogical_value` together with the optional `rl_pair` field before deciding whether to emit an RL training example

### Requirement: Processed document markdown is persisted to LanceDB

The system MUST vectorize the canonical `rag_markdown` generated by `worker-pdf`, `worker-html`, and `worker-jar` and append the resulting vector record to a LanceDB dataset stored at `gs://<PROJECT_ID>-rag-markdown-bucket/vectordb`.

#### Scenario: A processed document is routed into the vector store
- **WHEN** a document worker successfully produces a canonical `rag_markdown` payload
- **THEN** it requests an embedding for that Markdown content
- **AND** it appends the embedding, source text, and associated document metadata to the LanceDB dataset in the markdown bucket

### Requirement: Only high-value instructional pairs are appended to the RL dataset

The system MUST append RL fine-tuning examples to `gs://<PROJECT_ID>-rag-markdown-bucket/rl_dataset.jsonl` only when the teacher payload includes an eligible `rl_pair`.

#### Scenario: A processed document yields an eligible RL pair
- **WHEN** the teacher payload includes a populated `rl_pair`
- **AND** `pedagogical_value` is greater than 7
- **THEN** the worker serializes that pair as one JSON object line
- **AND** it appends the line to `rl_dataset.jsonl`

#### Scenario: A processed document is not eligible for RL extraction
- **WHEN** the teacher payload omits `rl_pair` or reports `pedagogical_value` less than or equal to 7
- **THEN** the worker does not append a new line to `rl_dataset.jsonl`

### Requirement: Workers log raw teacher model output in debug mode

The system MUST enable `worker-pdf`, `worker-html`, and `worker-jar` to log the raw, unmodified string output from the teacher model when a `DEBUG_LLM_OUTPUT` environment variable is set to `true`.

#### Scenario: A worker runs with debug mode enabled
- **GIVEN** `DEBUG_LLM_OUTPUT` is `true`
- **WHEN** the worker receives a response from the teacher model
- **THEN** it prints the raw string response to standard output before JSON parsing

#### Scenario: A worker runs with debug mode disabled
- **GIVEN** `DEBUG_LLM_OUTPUT` is not `true`
- **WHEN** the worker receives a response from the teacher model
- **THEN** it does not print the raw string response
